### Тест 1
#### Проверка обработки нулевой суммы перевода (0 ₽)
#### Цель теста: Убедиться, что система корректно обрабатывает попытку перевода нулевой суммы
#### Шаги выполнения:
1. Открыт URL: `http://localhost:8000/?balance=60000&reserved=1000`
2. Нажата область рублевого счета
3. Введен валидный номер карты: `1111 1111 1111 1111`
4. Указана сумма перевода: `0 ₽`
5. Выполнена попытка перевода
#### Желаемый результат
- Система отклоняет операцию
- Отображается сообщение: "Сумма перевода должна быть больше 0 ₽"
- Баланс и резерв остаются без изменений
#### Фактический результат
❌ Перевод выполнен с суммой 0 ₽  
❌ Отображено ложное уведомление об успешной операции  
![[Pasted image 20250614151052.png]]
#### Статус теста
**Failed**  
Критическая уязвимость: система принимает бессмысленные транзакции
#### Комментарии
1. **Проблемы:**
- Нарушение бизнес-логики (транзакции 0 ₽ не имеют смысла)
- Риск создания мусорных операций в истории переводов
- Ложное уведомление вводит пользователя в заблуждение
2. **Рекомендации:**
- Добавить валидацию: `Сумма > 0`
- Блокировать кнопку перевода при нулевом значении
- Заменить уведомление на ошибку: "Введите сумму больше нуля"
  
**Приоритет исправления:** Высокий (нарушение финансовой логики)

✅ Все три перевода выполнены

✅ Комиссия для каждого: `500 ₽`

**Passed**

2. **Рекомендации:**

2. Нажать на рублевый счет
3. Ввести номер карты: `5111 1111 1111 1111
4. Выполнить перевод: `2000 ₽

- Очистить поле суммы
- Ввести новую сумму: `3000 ₽`

✅ Переводы выполнены  
❌ Баланс корректно уменьшен на `5500 ₽` суммарно  
✅ Номер карты остался в поле для новых операций
#### Статус теста
**Passed**

1. **Практическое применение:**
